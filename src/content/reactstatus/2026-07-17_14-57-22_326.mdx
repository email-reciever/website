**IN BRIEF:**

*   ⭐ Guillermo Rauch [just announced on 𝕏](https://react.statuscode.com/link/188088/5d0bbb4a61) that Vercel has hired Pete Hunt and Nick Schrock (co-creator of GraphQL). Pete, an early and influential React team member, will be leading the Next.js project.
    
*   [Vercel is formalizing a predictable, monthly Next.js security release schedule](https://react.statuscode.com/link/188089/5d0bbb4a61) with the first coming in just a few days.
    
*   Cloudflare has entirely rebuilt [its blog](https://react.statuscode.com/link/188090/5d0bbb4a61) on top of [EmDash](https://react.statuscode.com/link/188091/5d0bbb4a61), the [Astro](https://react.statuscode.com/link/188092/5d0bbb4a61)\-powered 'spiritual successor to WordPress' it unveiled (genuinely) on April 1.
    

[I've Maintained Linaria for Six Years: Here's Why I Built Something New](https://react.statuscode.com/link/188093/5d0bbb4a61 "dx-styles.dev") — _"Runtime CSS-in-JS is winding down,"_ says Linaria's maintainer, who has now built [dx-styles](https://react.statuscode.com/link/188094/5d0bbb4a61), a zero-runtime compile-time alternative (for which there's also a [Linaria migration guide](https://react.statuscode.com/link/188095/5d0bbb4a61)).

Anton Evzhakov

[Cache Components in Next.js, Explained](https://react.statuscode.com/link/188096/5d0bbb4a61 "certificates.dev") — A walkthrough of Next.js 16's dynamic-by-default caching model, covering `use cache`, `cacheLife`, `cacheTag`, and how it powers Instant Navigations.

Aurora Scharff

📺 [Jack Herrington Can’t Think of a Reason to Choose Next.js Anymore](https://react.statuscode.com/link/188097/5d0bbb4a61) – Jack explains why he's all-in on the TanStack approach, as well as his experiences building with AI, in this 45-minute interview. Ankita Kulkarni

📺 [The Framework Wars Are Over: Why No One Dethroned React](https://react.statuscode.com/link/188098/5d0bbb4a61) – Kent's take in 11 minutes. Kent C. Dodds

📄 [Building a ChatGPT-Style AI Chat App in React Native](https://react.statuscode.com/link/188099/5d0bbb4a61) – That also uses RAG to answer domain-specific queries. Dave Mkpa-Eke (Margelo)

🛠  Code, Tools & Libraries

[![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/bjmfrkg2i2mujsoag2zb.jpg)](https://react.statuscode.com/link/188100/5d0bbb4a61)

[`shadcn/typeset`: A Complete Typography System for HTML](https://react.statuscode.com/link/188100/5d0bbb4a61 "ui.shadcn.com") — A drop-in typography system for rendered HTML (such as Markdown output) packaged as a single CSS file with one wrapper class. Rather than exposing dozens of typography settings, it's driven by a few CSS variables. There's a [Typeset builder](https://react.statuscode.com/link/188101/5d0bbb4a61) to help you customize it and generate code to get started.

shadcn Team

[DSSSP: Audio Equalization and Filter Visualization Components](https://react.statuscode.com/link/188102/5d0bbb4a61 "dsssp.io") — One for the audio folks. SVG-based components to visualize things like frequency response graphs and control audio filters. There are [some live demos](https://react.statuscode.com/link/188103/5d0bbb4a61).

Alex Strelets

[react-dropzone 17.0: Create File Drag-and-Drop Zones](https://react.statuscode.com/link/188104/5d0bbb4a61 "react-dropzone.js.org") — A mature library offering a hook-based approach to making HTML-compliant drag-and-drop zones for files dropped onto pages. The docs include [a basic demo with code](https://react.statuscode.com/link/188105/5d0bbb4a61).

Param Aggarwal

[React Mosaic 7.0: A Component Tiling 'Window Manager'](https://react.statuscode.com/link/188106/5d0bbb4a61 "github.com") — Provides an API to organize and tile React components across a view. [v7.0](https://react.statuscode.com/link/188107/5d0bbb4a61) makes big layout engine changes and targets React 16-19 with a full ESM + CJS build. [Demo here.](https://react.statuscode.com/link/188108/5d0bbb4a61)

Kevin Verdieck

[brainless: Claude Code, Codex, and Grok UIs as `shadcn` Components](https://react.statuscode.com/link/188109/5d0bbb4a61 "brainless.swerdlow.dev") — If, for some reason, you want to [recreate a TUI coding agent experience](https://react.statuscode.com/link/188110/5d0bbb4a61) on the Web, render [Claude-style diffs](https://react.statuscode.com/link/188111/5d0bbb4a61) or show off agent examples in docs, perhaps?

Ben Swerdlow

*   [react-fontawesome 3.4](https://react.statuscode.com/link/188112/5d0bbb4a61) – The official library to use [Font Awesome](https://react.statuscode.com/link/188113/5d0bbb4a61) icons in React apps adds support for [Font Awesome 7.3's](https://react.statuscode.com/link/188114/5d0bbb4a61) new animations.
    
*   [Preact 11.0 Beta 2](https://react.statuscode.com/link/188115/5d0bbb4a61) – _"One of the last betas for Preact 11,"_ says maintainer Jovi De Croock. There's already [a migration guide](https://react.statuscode.com/link/188116/5d0bbb4a61) ready to use.
    
*   [use-local-storage-state 20.0](https://react.statuscode.com/link/188117/5d0bbb4a61) – A hook that persists data in `localStorage`.
    
*   [react-dropdown 2.0](https://react.statuscode.com/link/188118/5d0bbb4a61) – Simple accessible dropdown component.
    
*   [react-joyride 3.2](https://react.statuscode.com/link/188119/5d0bbb4a61) – Create guided tours within your React apps.
    
*   [Travels 1.4](https://react.statuscode.com/link/188120/5d0bbb4a61) – Fast, reusable patch-based undo/redo mechanism.
    
*   [React Native Skia 2.9](https://react.statuscode.com/link/188121/5d0bbb4a61)

📊  Monitoring package popularity

[![](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/wc2von9kring79sp9kmv.jpg)](https://react.statuscode.com/link/188124/5d0bbb4a61)

Did you know TanStack maintains [an npm stats monitoring page](https://react.statuscode.com/link/188124/5d0bbb4a61) you can use to graph the popularity of any npm packages over time?

The default view shows how TanStack's collection of libraries fares against other popular sets from React Router, Next.js, and Angular, but you can also pick from other pre-defined groups like 'data grids' or 'state management', or just specify your own packages to compare.
