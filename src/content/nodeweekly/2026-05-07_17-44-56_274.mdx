📢  Node 26.1 Lands FFI Support

[Node.js 26.1.0 (Current)](https://nodeweekly.com/link/184889/0a4070b383) quickly followed Node.js 26 with a single notable feature: **an experimental `node:ffi` module.**

FFI ([Foreign Function Interface](https://nodeweekly.com/link/184917/0a4070b383)) is a mechanism to directly interface with libraries written in other languages (most commonly those exposing a C ABI).

FFI support has been _complicated_ in Node over the years. [node-ffi](https://nodeweekly.com/link/184918/0a4070b383) was an early option, but quite fragile. [Koffi](https://nodeweekly.com/link/184919/0a4070b383) offers a more robust take on the idea and continues to work well today, but wasn't adopted as an official, core solution.

Node settled on the [Node-API / NAPI approach](https://nodeweekly.com/link/184920/0a4070b383) of writing glue code to bridge the Node / C++ gap, an approach that's now universal in Node libraries with native dependencies.

With [this work](https://nodeweekly.com/link/184921/0a4070b383) now in Node 26.1 behind a `--experimental-ffi` flag, it seems we're finally getting a core, officially supported FFI solution. It integrates with Node's [permissions model](https://nodeweekly.com/link/184922/0a4070b383) too (and [it's fast](https://nodeweekly.com/link/184924/0a4070b383)).

Here's a basic (macOS only) example I just made to show off the basic shape of the support and how easy it is, though with only a single, simple call:

* * *

```js
import &lcub; dlopen } from 'node:ffi';
const path = '/usr/lib/libSystem.B.dylib';

const &lcub; functions } = dlopen(path, &lcub;
  sqrt: &lcub; parameters: ['f64'], result: 'f64' },
});

console.log(functions.sqrt(2));  // 1.414213...
```

* * *

I also tried SQLite 3 with some success, and am looking forward to giving [Raylib 6.0](https://nodeweekly.com/link/184923/0a4070b383) a go as an example of a far more complex C library.

My fingers are crossed that Node finds a way to make this stable, so we can all benefit from more direct use of native dependencies in Node in future 😄
