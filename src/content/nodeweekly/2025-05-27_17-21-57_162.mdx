This post summarizes why the official response is no, for now.

⚡ [Announcing TypeScript Native Previews](https://nodeweekly.com/link/169646/0a4070b383 "devblogs.microsoft.com") — Earlier this year, Anders Hejlsberg teased [a 10x faster TypeScript](https://nodeweekly.com/link/169647/0a4070b383) through efforts being made to port the TypeScript compiler to Go, enabling it to be compiled and run natively, as well as take advantage of more concurrency. You can now try it for yourself.

Microsoft

**IN BRIEF:**

*   The _Glitch_ platform, a popular way to build and host Node-backed webapps, is [shutting down its app hosting functionality](https://nodeweekly.com/link/169648/0a4070b383) – it seems uncertain quite what Glitch will or won't be after July. Former Glitch employee Keith Kurson [writes more about it here.](https://nodeweekly.com/link/169649/0a4070b383)
    
*   [Astro 5.8](https://nodeweekly.com/link/169650/0a4070b383) has been released and drops support for all Node versions prior to v18.20.8. _Angular 20_, due later this week, is also set to drop Node 18 support entirely.
    
*   You have just three more days to [take the latest Node.js Next 10 survey.](https://nodeweekly.com/link/169651/0a4070b383)
    
*   [Node v24.1 (Current)](https://nodeweekly.com/link/169652/0a4070b383) has been released.
    
*   [Node v22.16.0 (LTS)](https://nodeweekly.com/link/169653/0a4070b383) has also been released, notably adding `node.config.json` support.
    
*   Seokho Song writes about [becoming the latest V8 committer.](https://nodeweekly.com/link/169654/0a4070b383)
    

[Six Ways Slack, Notion, and VS Code Improved Electron App Performance](https://nodeweekly.com/link/169655/0a4070b383 "palette.dev") — An experienced Electron app developer shares a guide to Electron performance optimization and how to get the most out of your apps.

Amila Welihinda

[A Developer’s Guide to Migrating from API Keys to OAuth 2.1](https://nodeweekly.com/link/169656/0a4070b383 "www.scalekit.com") — Many still rely on API keys for service-to-service auth. But with MCP making OAuth 2.1 mandatory, it’s no longer just a “nice-to-have.” [Here's what’s changing](https://nodeweekly.com/link/169656/0a4070b383) and what it means for your architecture.
