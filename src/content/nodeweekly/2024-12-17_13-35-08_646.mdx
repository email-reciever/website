2024 has been a solid year for Node, with the [release of Node.js v22](https://nodeweekly.com/link/163510/0a4070b383) back in April ([now in LTS](https://nodeweekly.com/link/163511/0a4070b383)) and [v23 in October.](https://nodeweekly.com/link/163512/0a4070b383) Node also got a new mascot this year _(above)_, [Express 5.0](https://nodeweekly.com/link/163513/0a4070b383) arrived, Node has [dabbled with type stripping](https://nodeweekly.com/link/163514/0a4070b383) to better support TypeScript, and [a native SQLite module](https://nodeweekly.com/link/163515/0a4070b383) has been under active development. _Fingers crossed for an interesting 2025!_

[Node v23.4.0 (Current) Released](https://nodeweekly.com/link/163516/0a4070b383 "nodejs.org") — There are new `--trace-env`, `--trace-env-js-stack` and `--trace-env-native-stack` options to track environment variable use by a script. Also `assert.partialDeepStrictEqual` has been added as a way to more quickly assert _certain_ properties are present in an object – you don’t have to supply them all.

Antoine du Hamel

🥇 Top Node Items of 2024

From an epic 1694 links we included in _Node Weekly_ this year, we're going to boil it down to the top 8 you clicked on the most:

1. [A Guide to Reading and Writing Node.js Streams](https://nodeweekly.com/link/163526/0a4070b383 "blog.platformatic.dev") — I'm not surprised to see this as our most popular link of the year. While [streams](https://nodeweekly.com/link/163527/0a4070b383) are a long standing Node feature, they're commonly misunderstood, and Matteo did a good job of covering their benefits, use cases, and more.

Matteo Collina

2. [The Nine Node Pillars](https://nodeweekly.com/link/163528/0a4070b383 "www.platformatichq.com") — An interesting list of self-proclaimed 'principles for doing Node.js right in enterprise environments' written by a group of prolific Node contributors. It acts as a checklist to identify gaps in your current practices, particularly when building larger scale apps.

Snell, Venditto, Dawson, Collina, et al.

[Automate Embedding Creation and Syncing in Postgres with pgai Vectorizer](https://nodeweekly.com/link/163529/0a4070b383 "github.com") — 🚀 pgai Vectorizer automates embedding creation and syncing in Postgres with one SQL command—keeps embeddings updated, no tools needed.

Timescale sponsor

3. [10 Modern Node.js Runtime Features to Start Using in 2024](https://nodeweekly.com/link/163530/0a4070b383 "snyk.io") — It's not just Bun and Deno adding new features by the dozen over the past couple of years, there's plenty of new stuff in Node too. How many did you use in 2024?

Liran Tal

4. [Introducing Express v5](https://nodeweekly.com/link/163513/0a4070b383 "expressjs.com") — We first spied the release of Express.js v5 in September but the official blog post eventually dropped in October and explained the broad plan for Node's long standing Web framework. v5 remains a '_bleeding edge_' release with work still required in areas of security (such as [this security audit](https://nodeweekly.com/link/163531/0a4070b383)) and overall processes.

Wes Todd (Express)

5. [TypeSpec: A TypeScript-Inspired Way to Define APIs](https://nodeweekly.com/link/163532/0a4070b383 "typespec.io") — A language for concisely describing cloud service APIs and generating other API description languages (e.g. OpenAPI), client and service code, docs, etc. [v0.63](https://nodeweekly.com/link/163533/0a4070b383) has just been released, by the way.

Microsoft

💡 Nate Totten's '[Bringing Types to APIs with TypeSpec](https://nodeweekly.com/link/163534/0a4070b383)' will give you an idea of the practicalities of using TypeSpec.

6. [zx v8.0: Google's Way to Write Shell Scripts with Node](https://nodeweekly.com/link/163535/0a4070b383 "github.com") — A long standing way to make shell scripting a more pleasant experience with Node. zx provides useful wrappers around `child_process`, escapes arguments and gives sensible defaults. v8.0 made zx _20x smaller_, faster, makes it easier to kill processes, pass input to commands, and more. It remains the latest major version, with [v8.2.4](https://nodeweekly.com/link/163536/0a4070b383) landing a few weeks ago.

Google

7. [The Node.js Best Practices List: 2024 Edition](https://nodeweekly.com/link/163537/0a4070b383 "github.com") — An in-depth guide for Node developers that we link to most years. Divided into eight sections and updated regularly, it digs into areas from error handling and code style to Docker and security practices.

Yoni Goldberg

8. [Announcing AdonisJS v6](https://nodeweekly.com/link/163538/0a4070b383 "adonisjs.com") — A TypeScript-first backend Web framework with [amazing documentation](https://nodeweekly.com/link/163539/0a4070b383) and packed with features out the box. v6 was a huge step forward that moved to using ESM by default and remains the latest major version, despite being released in January.

Harminder Virk

[Node v23.4.0（当前）发布](https://nodeweekly.com/link/163516/0a4070b383 "nodejs.org") — 新增 `--trace-env`、`--trace-env-js -stack` 和 `--trace-env-native-stack` 选项用于跟踪脚本使用的环境变量。此外，还添加了“assert.partialDeepStrictEqual”，作为一种更快速地断言对象中存在*某些*属性的方法 - 您不必提供全部属性。
