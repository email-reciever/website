[npm v12 to Stop Running Install Scripts by Default](https://nodeweekly.com/link/186406/0a4070b383 "github.blog") — After a year of supply chain attacks, npm v12 will no longer execute `preinstall`, `install`, or `postinstall` scripts, unless you allow them with a new `npm approve-scripts` workflow. **You can prepare today** by upgrading to [npm 11.16.0](https://nodeweekly.com/link/186407/0a4070b383) which prints warnings about anything v12 _would_ block.

GitHub

🔒 `npm install` will no longer resolve git or remote URLs by default either – these can be switched on using `--allow-git` and `--allow-remote`.

Luciano goes deep into how it’s happening, when, and why.

Luciano Mammino

**IN BRIEF:**

*   🔒 [New versions of Node.js v26.x, 24.x, and 22.x](https://nodeweekly.com/link/186409/0a4070b383) will be released on, or shortly after, Wednesday, June 17 to address security issues – the highest severity of which is 'HIGH'.
    
*   🔒 As the [npm worms/supply chain attacks continue](https://nodeweekly.com/link/186410/0a4070b383), I was intrigued to see [this vulnerability report from 2016](https://nodeweekly.com/link/186411/0a4070b383) which predicted our current woes.
    
*   The [Drizzle ORM](https://nodeweekly.com/link/186412/0a4070b383) team is [𝕏 unable to publish new releases to npm](https://nodeweekly.com/link/186413/0a4070b383) due to the registry's 100MB metadata limit. Their package currently has [1,397 versions published](https://nodeweekly.com/link/186414/0a4070b383) and will need some cleanup.
    
*   In less alarming registry news, an npm tooling bug briefly [marked several packages with one-character names as ‘security holders’](https://nodeweekly.com/link/186415/0a4070b383) (including `c`, `i`, `n` and `x`) by mistake. A rollback is underway.
    

📊 [ESM Provision by Popular npm Packages Sees a Big Jump](https://nodeweekly.com/link/186416/0a4070b383 "github.com") — Titus’s twice-yearly look at what popular packages ship finds 38% now expose ESM (and 16% are ESM _only_), up from 33.4% six months ago.

Titus Wormer

[Node-RED 5.0: The Biggest Editor Overhaul in the Project's History](https://nodeweekly.com/link/186421/0a4070b383 "nodered.org") — The Node.js _and_ 'node'-based low-code environment gets _"the biggest change to the editor experience in \[its\] history"_: updated sidebars, dark theme _(above)_, pausable debug output, and the ability to call Link nodes from Function nodes.

This Rust-based tool audits and rewrites your package manager config, disabling install scripts, enforcing cooldowns, and blocking provenance downgrades.

Arnica

*   ⭐ [Commander.js 15.0](https://nodeweekly.com/link/186425/0a4070b383) – The popular CLI framework is now ESM-only. v14 will be maintained till May 2027 in case you have trouble migrating.
    
*   🎬 [NodeAV 6.0](https://nodeweekly.com/link/186426/0a4070b383) – Direct access to FFmpeg's C APIs with minimal abstractions for working with codecs, filters, streams, and more.
    
*   [Neutralinojs 6.8](https://nodeweekly.com/link/186427/0a4070b383) – The lightweight cross-platform app framework gets improved native file drag and drop, and a new default window policy to handle `target="_blank"` links better.
    
*   [Juice 12.1](https://nodeweekly.com/link/186428/0a4070b383) – Automattic's library to inline CSS stylesheets into HTML. A common HTML email production task, [as used by](https://nodeweekly.com/link/186429/0a4070b383) MJML and `node-email-templates`.
    
*   [Cucumber.js 13.0](https://nodeweekly.com/link/186430/0a4070b383) – The BDD test framework rebuilds its parallel runtime atop worker threads.
    
*   [Kanel 4.0](https://nodeweekly.com/link/186431/0a4070b383) – Generate TypeScript types from a live Postgres database.
    
*   [Envalid 8.2](https://nodeweekly.com/link/186432/0a4070b383) – Environment variable validation library.
