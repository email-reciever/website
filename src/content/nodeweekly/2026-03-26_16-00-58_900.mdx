🟦 TypeScript 6.0 for Node developers

[Announcing TypeScript 6.0](https://nodeweekly.com/link/182809/0a4070b383 "devblogs.microsoft.com") — TypeScript 6.0 is designed to bridge the way from its [self-hosted compiler](https://nodeweekly.com/link/182843/0a4070b383) to [the Go-powered native compiler future](https://nodeweekly.com/link/182844/0a4070b383) of TypeScript 7.0 (which, we're told, is almost ready to go, too).

As always, the release post is _huge_ but here are the things that jumped out to me as affecting Node developers the most:

*   [**`types` now defaults to `[]`**](https://nodeweekly.com/link/182845/0a4070b383). TypeScript will no longer pull in potentially thousands of type declarations at build time. Great for performance, but... if you were relying on it, your project will break! `"types": ["node"]` or `"types": ["*"]` may be your friend at first.
    
*   [**`strict` is now `true` by default.**](https://nodeweekly.com/link/182846/0a4070b383) Many of you will already be working in strict mode, but if not, get ready for a flood of new type errors unless you explicitly set it to `false`.
    
*   [**`--moduleResolution node` is deprecated.**](https://nodeweekly.com/link/182847/0a4070b383) If you're targeting Node, you'll now use `--moduleResolution nodenext` which reflects how modern versions of Node resolve modules.
    
*   **`module` now defaults to `esnext`.** CommonJS projects must now set `"module": "commonjs"` explicitly.
    
*   [**`#/` subpath imports now work.**](https://nodeweekly.com/link/182848/0a4070b383) Available in Node [since v25.4](https://nodeweekly.com/link/182849/0a4070b383) (and backported into LTS releases), but now in TypeScript too.
    
*   [**`rootDir` now defaults to `.`**](https://nodeweekly.com/link/182850/0a4070b383) If you have source files deeper than your `tsconfig.json` and were relying on TypeScript to infer a common root directory, you now need to explicitly set `rootDir`.
    
*   [**`--esModuleInterop false` and `--allowSyntheticDefaultImports false` no longer work.**](https://nodeweekly.com/link/182851/0a4070b383) So if you're still using syntax like `import * as express from "express"` it won't work, you'd need to use `import express from "express"`.
    

In short, TypeScript 6.0 tightens defaults across the board and throws enough changes into the works that many projects will need at least a tsconfig tweak.. good luck with your upgrades! 🤞
