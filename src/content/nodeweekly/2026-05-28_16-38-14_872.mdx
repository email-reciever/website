[A Production Playbook for Node.js Stream Leaks](https://nodeweekly.com/link/185782/0a4070b383 "frontendmasters.com") — You used `pipeline()` but your streams still leak in production. This guide covers five failure modes that sail past tests and code review, then break under real traffic, from unnoticed disconnects to timeouts that kill the response but leave an upstream fetch running. Useful if you run Node services that stream under load.

Durgesh Rajubhai Pawar

💡 Durgesh [covers how backpressure works](https://nodeweekly.com/link/185783/0a4070b383) (and can fail) in a prior part, if you want more of an intro to why streams don't protect you from memory exhaustion on their own.

[![](https://res.cloudinary.com/cpress/image/upload/c_limit,w_480,h_480,q_auto/copm/d9fd428c.jpg)](https://nodeweekly.com/link/185781/0a4070b383)

[Build Durable AI Agents That Won't Break in Production (Webinar)](https://nodeweekly.com/link/185781/0a4070b383 "orkes.io") — Learn the open-source 5-layer stack and run durable AI agents with Agentspan across LangGraph, OpenAI, and Google SDKs.

Orkes sponsor

[Deno 2.8 Hits 76% Node.js Compatibility](https://nodeweekly.com/link/185820/0a4070b383 "deno.com") — A significant release for the JS runtime created by Node’s original creator. Node.js compatibility jumps from 42% to 76.4% (higher than Bun). v2.8 also drops the `npm:` prefix requirement when adding/installing packages (so `deno install` is now a drop-in for `npm install`/`pnpm install` etc. even in existing Node projects).

Bartek Iwańczuk

**IN BRIEF:**

*   [npm 11.16.0 has shipped](https://nodeweekly.com/link/185821/0a4070b383) with initial (advisory warning only) support for an _opt-in_ install-script policy using `allowScripts`. [More details here.](https://nodeweekly.com/link/185822/0a4070b383)
    
*   The Node team is [reconsidering the ABI stability requirement for V8 updates](https://nodeweekly.com/link/185788/0a4070b383) in major Node releases after Node 26 was delayed for V8 14.6.
    
*   🇮🇹 [NodeConf EU 2026](https://nodeweekly.com/link/185789/0a4070b383) takes place this September in Italy. The site has had a redesign and [the CFP is still open.](https://nodeweekly.com/link/185790/0a4070b383)
    
*   [macOS on Intel (x64) has become a 'tier 2' platform](https://nodeweekly.com/link/185791/0a4070b383) for Node 26 meaning if binary builds can't be made on the platform, it won't block releases for the others. You can learn more about [Node's support tiers here](https://nodeweekly.com/link/185792/0a4070b383).
    

[Node.js 24.16.0 (LTS) Released](https://nodeweekly.com/link/185785/0a4070b383 "nodejs.org") — If Node 26 is too cutting edge and LTS is more your vibe, there are still new features to be had including `crypto.randomUUIDv7()`, [`util.styleText`](https://nodeweekly.com/link/185786/0a4070b383) accepting hex colors, and [test-order randomization](https://nodeweekly.com/link/185787/0a4070b383) in Node’s test runner.

Antoine du Hamel

[Migrating an Express App to Next.js Using AI Agents](https://nodeweekly.com/link/185793/0a4070b383 "javascript.plainenglish.io") — James (who works on [Antigravity](https://nodeweekly.com/link/185794/0a4070b383) at Google) demonstrates how to modernize a legacy Express.js app to Next.js App Router and TypeScript. The [skills](https://nodeweekly.com/link/185795/0a4070b383) built to do this have been shared for use by anyone.

James O'Reilly (Google)

[Using AI to Write Better Code More Slowly](https://nodeweekly.com/link/185796/0a4070b383 "nolanlawson.com") — A prolific JavaScript developer says LLMs aren’t just for pumping out bad code quickly, they can indeed help you write higher quality code more _slowly_.

Nolan Lawson

Dolan Miu

*   [TypeSQL 0.24](https://nodeweekly.com/link/185811/0a4070b383) – Generate typesafe TypeScript APIs from SQL statements. Supports MySQL, Postgres, and SQLite3.
