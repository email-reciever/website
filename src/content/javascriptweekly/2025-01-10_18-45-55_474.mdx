**IN BRIEF:**

*   🥊 The Deno vs Oracle fight over the JavaScript™ trademark continues as Oracle has informed Deno [they're not going to voluntarily relinquish the mark.](https://javascriptweekly.com/link/164152/d076523b1d) Deno's job now is to show how JavaScript has long been used as a generic term and hasn't been controlled by Oracle.
    
*   The Express.js team has posted [about the project's recent revival and its plans to push Express.js forward in 2025.](https://javascriptweekly.com/link/164153/d076523b1d)
    
*   ❄️ The 'WinterCG' _Web Interoperable Runtimes Community Group_, an effort to promote standards around runtime interoperability of JavaScript runtimes, [has moved to Ecma International](https://javascriptweekly.com/link/164154/d076523b1d) and is now known as [WinterTC (TC55).](https://javascriptweekly.com/link/164155/d076523b1d)

**RELEASES:**

*   [pnpm 10](https://javascriptweekly.com/link/164159/d076523b1d) – The efficient `npm` alternative no longer runs lifecycle scripts of dependencies for security reasons, hashing algorithms have been upgraded to SHA256, and lots of minor tweaks.
    
*   [Bun v1.1.43](https://javascriptweekly.com/link/164160/d076523b1d) – The high-performance runtime gets first class S3 support, an HTML bundler, and can output V8 heap snapshots (which is quite something as Bun uses JavaScriptCore, not V8).
    
*   🔠 [Tesseract.js 6.0](https://javascriptweekly.com/link/164161/d076523b1d) – The popular pure JS multilingual OCR library has resolved a variety of memory leak issues.
    
*   [Docusaurus 3.7](https://javascriptweekly.com/link/164162/d076523b1d) – The popular docs-oriented site generator goes full React 19.
    
*   [Node.js v22.13.0 (LTS)](https://javascriptweekly.com/link/164163/d076523b1d) – The [permission model system](https://javascriptweekly.com/link/164164/d076523b1d) is now stable.

📄 [Play Tetris in a PDF File](https://javascriptweekly.com/link/164212/d076523b1d "th0mas.nl") — (direct link to PDF)

I'll let you decide if this one is fun or frightening! Whether or not this will work depends on your PDF reader or browser support, but it works with Chrome and Firefox, at least.

The PDF document format supports embedded JavaScript and this experiment uses it to implement a game of Tetris. The developer, Thomas Rinsma, has [used Python to output the PostScript that includes the game's JavaScript.](https://javascriptweekly.com/link/164213/d076523b1d) Couple that with the fact many browser PDF renderers are _themselves_ implemented in JavaScript (e.g. [PDF.js](https://javascriptweekly.com/link/164214/d076523b1d)) and you have a veritable Matryoshka doll of technologies at play here.
