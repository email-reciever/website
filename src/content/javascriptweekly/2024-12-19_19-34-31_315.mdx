🗓️ What Happened to JavaScript in 2024

The JavaScript world has had a busy 2024, as you'd expect for [the world's most used programming language](https://javascriptweekly.com/link/163779/d076523b1d) (despite the threat of [being split into two](https://javascriptweekly.com/link/163801/d076523b1d)). We looked back over the year and remembered some things that occurred:

*   In February, the React team posted [a mega 'React Labs' update](https://javascriptweekly.com/link/163802/d076523b1d) which framed the whole of React's year, explaining the goals of the React Compiler and the eventual React 19 release (which [went stable](https://javascriptweekly.com/link/163803/d076523b1d) this month).
    
*   Alternative JavaScript runtimes had a great 2024, particularly Bun which introduced [Bun Shell](https://javascriptweekly.com/link/163804/d076523b1d), [Windows support](https://javascriptweekly.com/link/163805/d076523b1d) and the ability to [compile and run native C from JavaScript.](https://javascriptweekly.com/link/163807/d076523b1d) Deno had a big year too, with [the release of Deno 2](https://javascriptweekly.com/link/163808/d076523b1d) which included stronger Node.js/npm compatibility and package management tooling. Other systems continuing to progress this year include [Boa JS](https://javascriptweekly.com/link/163809/d076523b1d), [QuickJS](https://javascriptweekly.com/link/163810/d076523b1d), and [Porffor](https://javascriptweekly.com/link/163811/d076523b1d).
    
*   Over at TC39, [lots of language proposals were advanced](https://javascriptweekly.com/link/163812/d076523b1d) this year (and that's just a sample – [here's some more!](https://javascriptweekly.com/link/163813/d076523b1d)). The ES2025 future is bright.
    

[Boost Semantic Search with MongoDB Atlas Vector Search](https://javascriptweekly.com/link/163815/d076523b1d "fnf.dev") — Create embeddings, index them, and run semantic queries—just follow the quick start tutorial.

MongoDB sponsor

*   Deno decided to take the JavaScript trademark fight to Oracle and [formally filed a petition with the USPTO to cancel it.](https://javascriptweekly.com/link/163816/d076523b1d) We hope to see some developments here in 2025, but Oracle are prepared to defend it.
    
*   Not content to merely work on an edge platform, JS runtime, and fight Oracle, Deno also [unveiled JSR](https://javascriptweekly.com/link/163817/d076523b1d), a fresh attempt at providing a registry for JavaScript packages.
    
*   There were lots of big releases generally for major JavaScript projects including [Svelte v5](https://javascriptweekly.com/link/163819/d076523b1d), [Node.js v23.0](https://javascriptweekly.com/link/163820/d076523b1d), [Astro 5.0](https://javascriptweekly.com/link/163821/d076523b1d), [TypeScript 5.7](https://javascriptweekly.com/link/163822/d076523b1d), [Vite 6.0](https://javascriptweekly.com/link/163823/d076523b1d), [React Native 0.76](https://javascriptweekly.com/link/163824/d076523b1d), [Next.js 15](https://javascriptweekly.com/link/163825/d076523b1d), [React Router v7](https://javascriptweekly.com/link/163826/d076523b1d), [Rspack 1.0](https://javascriptweekly.com/link/163827/d076523b1d), [Vue.js 3.5](https://javascriptweekly.com/link/163828/d076523b1d), and [Angular 19](https://javascriptweekly.com/link/163829/d076523b1d).
    
*   The OG of JavaScript libraries [jQuery 4.0 went into beta](https://javascriptweekly.com/link/163830/d076523b1d) too! We keep our fingers crossed for a jQuery 4.0 final release in 2025.. ;-)

🗓️ 2024 年 JavaScript 发生了什么

正如您对[世界上最常用的编程语言](https://javascriptweekly.com/link/163779/d076523b1d)所期望的那样，JavaScript 世界度过了忙碌的 2024 年（尽管存在\[被一分为二]的威胁）( https://javascriptweekly.com/link/163801/d076523b1d))。回顾这一年，我们想起了一些发生的事情：

* 二月份，React 团队发布了[大型“React Labs”更新](https://javascriptweekly.com/link/163802/d076523b1d)，其中框架了整个 React 的一年，解释了 React 编译器的目标和最终的目标React 19 版本（[稳定](https://javascriptweekly.com/link/163803/d076523b1d) 月）。

* 替代 JavaScript 运行时在 2024 年表现出色，特别是 Bun，它引入了 [Bun Shell](https://javascriptweekly.com/link/163804/d076523b1d)、\[Windows 支持]\(https://javascriptweekly.com/link/163805/ d076523b1d）以及[编译并运行本机 C 的能力JavaScript.](https://javascriptweekly.com/link/163807/d076523b1d) Deno 也度过了重要的一年，[Deno 2 的发布](https://javascriptweekly.com/link/163808/d076523b1d) 其中包括更强的 Node.js/npm 兼容性和包管理工具。今年继续取得进展的其他系统包括 [Boa JS](https://javascriptweekly.com/link/163809/d076523b1d)、[QuickJS](https://javascriptweekly.com/link/163810/d076523b1d) 和 [Porffor ](https://javascriptweekly.com/link/163811/d076523b1d)。

* 在 TC39 上，今年[提出了很多语言提案](https://javascriptweekly.com/link/163812/d076523b1d)（这只是一个示例 - \[这里还有更多！]\(https://javascriptweekly. com/link/163813/d076523b1d))。 ES2025 的未来是光明的。

[使用 MongoDB Atlas 矢量搜索增强语义搜索](https://javascriptweekly.com/link/163815/d076523b1d "fnf.dev") — 创建嵌入、为其建立索引并运行语义查询 — 只需按照快速入门教程进行操作即可。

MongoDB 赞助商

* Deno 决定向 Oracle 提起 JavaScript 商标之争，并[正式向 USPTO 提交请愿书要求取消该商标](https://javascriptweekly.com/link/163816/d076523b1d) 我们希望在 2025 年看到一些进展，但甲骨文已准备好捍卫它。

* Deno 不满足于仅仅在边缘平台、JS 运行时上工作并对抗 Oracle，它还[推出了 JSR](https://javascriptweekly.com/link/163817/d076523b1d)，这是为 JavaScript 包提供注册表的全新尝试。
