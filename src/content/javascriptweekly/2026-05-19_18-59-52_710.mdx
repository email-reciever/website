RFC: [It’s Time for `npm` to Make Install Scripts Opt-In](https://javascriptweekly.com/link/185307/d076523b1d "github.com") — npm is the only major package manager that runs dependency install scripts (e.g. `postinstall`) by default, and they’ve become too much of a security weakness, says Jamie, who works for GitHub (maintainers of npm). This RFC features further discussion of the idea and the tradeoffs involved.

Jamie Magee

💡 [npq](https://javascriptweekly.com/link/185308/d076523b1d) is a tool that makes `npm install`s safer. It stands in front of `npm` and audits packages before installing them, including the presence of pre/post install scripts.

[![](https://res.cloudinary.com/cpress/image/upload/c_limit,w_480,h_480,q_auto/copm/f5067c40.png)](https://javascriptweekly.com/link/185306/d076523b1d)

[How Depot Built a CI Orchestrator on AWS Lambda](https://javascriptweekly.com/link/185306/d076523b1d "fandf.co") — Long-running CI orchestration without long-lived servers. Depot rebuilt their CI engine using AWS Lambda durable functions — stateful, callback-driven, and crash-recoverable. A deep dive into the run-workflow-job hierarchy powering Depot CI.

**IN BRIEF:**

*   😱 Dr. Axel Rauschmayer (JavaScript legend and former JS Weekly editor) has [taken his blog and JavaScript books off the Web](https://javascriptweekly.com/link/185312/d076523b1d) due to being overwhelmed by AI crawlers. You can, however, still [purchase his fantastic books here.](https://javascriptweekly.com/link/185313/d076523b1d)
    
*   The Bun saga continues. Despite once [playing down its significance](https://javascriptweekly.com/link/185314/d076523b1d), the [Rust-based rewrite of Bun has been merged](https://javascriptweekly.com/link/185315/d076523b1d), though there are [questions over the quality](https://javascriptweekly.com/link/185316/d076523b1d) of the AI-ported code. Much [discussion ensued on Hacker News.](https://javascriptweekly.com/link/185317/d076523b1d)
    
*   [The Deno team is teasing Deno 2.8](https://javascriptweekly.com/link/185318/d076523b1d), due to be released this week. Significant Node.js compatibility improvements, `import defer`, and TypeScript 6.0.3 support await.
    
*   The Chrome and Edge teams are working on [a new `&lt;install>` HTML element](https://javascriptweekly.com/link/185319/d076523b1d) for browsers to render a 'trusted install button' for PWAs.
    
*   The _Express.js_ project has [an all new look](https://javascriptweekly.com/link/185320/d076523b1d), including a new site, logo, and [improved docs.](https://javascriptweekly.com/link/185321/d076523b1d)
    

**RELEASES:**

*   [Angular 22 Release Candidate](https://javascriptweekly.com/link/185322/d076523b1d) – Final is due in early June. Expect [signal-based forms](https://javascriptweekly.com/link/185323/d076523b1d) and the [OnPush](https://javascriptweekly.com/link/185324/d076523b1d) change detection strategy becoming default.
    
*   [Bun 1.3.14](https://javascriptweekly.com/link/185325/d076523b1d) – The alternative JS runtime gets `Bun.Image`, more HTTP/2 and HTTP/3 support, and more Node compatibility improvements.
    
*   [ESLint Config Inspector 3.0](https://javascriptweekly.com/link/185326/d076523b1d) – A visual tool for inspecting and understanding your ESLint flat configs.
    
*   [TypeORM 1.0](https://javascriptweekly.com/link/185327/d076523b1d), [ESLint 10.4.0](https://javascriptweekly.com/link/185328/d076523b1d), [Relay 21.0](https://javascriptweekly.com/link/185329/d076523b1d), [Rolldown 1.0.1](https://javascriptweekly.com/link/185330/d076523b1d)
