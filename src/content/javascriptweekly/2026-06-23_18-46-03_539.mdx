[Desktop Apps With `deno desktop`](https://javascriptweekly.com/link/186910/d076523b1d "docs.deno.com") — Deno 2.9 (or the 'canary' build _now_) can turn JavaScript projects into self-contained apps on macOS, Windows, and Linux. Unlike Electron, you can opt to use the default OS WebView _or_ a bundled Chromium backend, plus you get cross-compilation and automatic support for apps built on frameworks like Next.js and SvelteKit.

The Deno Project

Greptile reviews each PR with full repo context, runs the code in a sandbox, and returns screenshots, logs, and traces as proof of what broke. Use Greptile in GitHub, GitLab, or from your terminal.

Greptile sponsor

[Babel 8.0 Released: The JavaScript Transpiler](https://javascriptweekly.com/link/186911/d076523b1d "babeljs.io") — It’s been _eight years_ since [Babel 7.0](https://javascriptweekly.com/link/186912/d076523b1d), and despite the rise of tools like SWC and Oxc, weekly downloads of the `@babel` packages have increased 380x since. v8.0 is a modernization release that goes ESM-only and targets ~ES2023 by default (vs ES5 before).

Babel Team

[TypeScript 7.0 Release Candidate](https://javascriptweekly.com/link/186913/d076523b1d "devblogs.microsoft.com") — A step closer for the Go-powered TypeScript compiler that promises ~10x faster build performance. _(I've seen people on social media getting very excited by this.)_

Daniel Rosenwasser (Microsoft)

**IN BRIEF:**

*   Cloudflare has added the ability to [create and deploy Workers on 'temporary accounts'](https://javascriptweekly.com/link/186914/d076523b1d). Ostensibly 'for AI agents', but you can use them yourself too, and get a Worker running for 60 minutes with no strings attached.
    
*   TC39 delegate Luca Casonato has made [a proposal to add built-in localized time formatting](https://javascriptweekly.com/link/186915/d076523b1d) to `&lt;time>`, akin to GitHub's [`&lt;relative-time>` element](https://javascriptweekly.com/link/186916/d076523b1d).
    
*   🤖 [MDN has introduced an official MCP server](https://javascriptweekly.com/link/186917/d076523b1d) to provide better access to MDN's up-to-date JS and frontend docs to coding agents.
    
*   The Bun team has proposed [adding shared-memory threads to JavaScriptCore](https://javascriptweekly.com/link/186918/d076523b1d) (a more traditional threading approach versus [worker threads](https://javascriptweekly.com/link/186919/d076523b1d)).
    

**RELEASES:**

*   ⭐ [Vite 8.1](https://javascriptweekly.com/link/186920/d076523b1d) – Adds experimental support for 'bundled dev mode' which radically speeds up dev server startup and full reloads on large apps. Also adds [WASM/ESM integration](https://javascriptweekly.com/link/186921/d076523b1d) support.
    
*   ⭐ [Astro 7.0](https://javascriptweekly.com/link/186922/d076523b1d) – The popular web framework gets faster build times, with `.astro` compilation and Markdown/MDX processing going through a Rust-powered pipeline. _Advanced Routing_ also gives you full control over the request pipeline.
    
*   [pnpm 11.7](https://javascriptweekly.com/link/186923/d076523b1d) and [11.8](https://javascriptweekly.com/link/186924/d076523b1d) – `pnpm install` gets a `--dry-run` option, `--frozen-store` adds support for read-only package stores, and more.
    
*   [Node.js v26.3.1 (Current)](https://javascriptweekly.com/link/186925/d076523b1d), [v24.17.0 (LTS)](https://javascriptweekly.com/link/186926/d076523b1d), and [v22.23.0 (LTS)](https://javascriptweekly.com/link/186927/d076523b1d)
