[JSIR: A High-Level IR for JavaScript from Google](https://javascriptweekly.com/link/183303/d076523b1d "discourse.llvm.org") — Google has open sourced a new tool ([JSIR](https://javascriptweekly.com/link/183304/d076523b1d)) and proposed an industry-standard IR ([Intermediate Representation](https://javascriptweekly.com/link/183305/d076523b1d) – if an AST tells you what the code _looks like_, an IR tells you what it _does_) for JavaScript. Already used at Google for analysis and code transformation, the underlying idea could form a foundation for a new generation of tooling.

Zhixun Tan (Google)

💡 Most devs won't feel the impact for a while, but this is the kind of groundwork that can lead to better linters, smarter bundlers, better refactoring tools, and so forth.

**RELEASES:**

*   [ESLint v10.2.0](https://javascriptweekly.com/link/183311/d076523b1d) – Adds support for language-aware rules through a new `meta.languages` property. `Temporal` is now also supported.
    
*   [Node.js 25.9.0 (Current)](https://javascriptweekly.com/link/183312/d076523b1d) – Adds `--max-heap-size` to set a max heap size for a process, and includes [`stream/iter`](https://javascriptweekly.com/link/183313/d076523b1d), a new experimental iterable streams API.
