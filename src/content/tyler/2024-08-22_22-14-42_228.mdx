1.  The Nuxt team just released a beta version of [Nuxt Scripts](https://click.convertkit-mail4.com/8kud0xdgv8aoh0rwnvmtluk9o2999h3/48hvh7ur8xmqkdhx/aHR0cHM6Ly9udXh0LmNvbS9ibG9nL251eHQtc2NyaXB0cw==), which provides a new and improved way of working with third-party scripts. 6.  [pkg.pr.new](https://click.convertkit-mail4.com/8kud0xdgv8aoh0rwnvmtluk9o2999h3/l2heh6u37xlvq5s6/aHR0cHM6Ly9naXRodWIuY29tL3N0YWNrYmxpdHotbGFicy9wa2cucHIubmV3) gives you continuous preview releases for your library, where each of your commits and PRs will trigger an instant preview release without publishing anything to NPM (it uses its own URLs instead). \[sponsored\]
    
7.  [Pragtical](https://click.convertkit-mail4.com/8kud0xdgv8aoh0rwnvmtluk9o2999h3/m2h7h6u8z4302gbm/aHR0cHM6Ly9wcmFndGljYWwuZGV2Lw==) is a new code editor that’s (wait for it) “practical _and_ pragmatic.” Who says developers can’t appreciate poetry?

Sunil Pai wrote about [Spatial Compute](https://click.convertkit-mail4.com/8kud0xdgv8aoh0rwnvmtluk9o2999h3/7qh7h2uo5n9pdpuz/aHR0cHM6Ly9zdW5pbHBhaS5kZXYvcG9zdHMvc3BhdGlhbC1jb21wdXRlLw==), which he defines as “writing code for a server in ONE file, and having different parts of it run in different parts of the world depending on what’s best for the user.” Cloudflare employees only want one thing and it’s disgusting.
